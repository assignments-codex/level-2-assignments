## Objectives

- Practice `try/catch/finally` and async error patterns.
- Guard `fetch` with `res.ok` checks and readable error messages.
- Add error handling to one real API call.

## Instructor Notes

- Show a minimal `fetchJson` helper (network -> HTTP -> JSON error paths).
- Demonstrate friendly messages and a simple retry or fallback.

## Hourly Breakdown

- **Hour 1:** Patterns (`try/catch/finally`, `res.ok`, JSON parse guard).
- **Hour 2:** Apply to an API of your choice (e.g., public data or classroom sample).
