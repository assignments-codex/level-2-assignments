**Theme:** Forms & semantic HTML; ARIA intro; semantic best practices
**Style:** Virtual, live‑coding while narrating choices for labels, names, and accessibility.

## Learning Goals

- Create an accessible HTML form with labels and useful attributes.
- Use semantic layout elements and explain why they help structure/SEO/accessibility.
- Add basic ARIA attributes only where appropriate (complement-not replace-native semantics).

## Live‑Coding Demo Checklist

- Start from a semantic shell; add a form with name/email/textarea/select + submit.
- Wire **labels** via `for`/`id`; show `required`, `placeholder`, and helpful `aria-*` where needed.
- Discuss when **not** to add ARIA (avoid redundancy).

## Common Pitfalls

- Inputs without labels; overusing ARIA instead of native elements.
- Poor field naming; missing `required`/`type="email"` for basic validation.

## Optional Extensions

- Add inline validation hints; mention color‑contrast considerations ().
