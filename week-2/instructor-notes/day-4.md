**Theme:** Common sections (hero, CTA, etc.); build a splash page; RGBA transparency
**Style:** Live‑coding a tiny landing page that layers text over backgrounds.

## Learning Goals

- Define common sections (hero/header/nav/content/CTA/footer) and when to use them.
- Build a small “splash” page with a hero area.
- Use **RGBA** to apply transparent overlays for readability.

## Live‑Coding Demo Checklist

- Create a hero section with heading/paragraph and a CTA button.
- Add RGBA backgrounds (e.g., dark overlay at 50%) and a semi‑transparent CTA background.
- Keep markup semantic and minimal; style with a dedicated CSS file.

## Common Pitfalls

- Using opacity on the container (affects children) instead of RGBA background color.
- Non‑semantic wrappers for styling only.

## Optional Extensions

- Add a simple mobile breakpoint to center/resize hero text.
