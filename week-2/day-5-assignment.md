## Objective

Integrate Week 2 skills in one small page using semantic structure, a simple form with labels/ARIA, Flexbox layout, and CSS box‑model overrides.

## Scope (keep it tight)

1. **Structure & Semantics:** header/nav • hero • content • CTA • footer.
2. **Form & Accessibility:** name/email/textarea + labels; use `required`; add a _minimal_ ARIA attribute only if needed.
3. **Layout:** Flexbox for either the header nav _or_ the CTA block; add one mobile breakpoint.
4. **Styling:** Demonstrate box‑model (padding/border/margin) on a visible element; override at least one user‑agent default (e.g., paragraph margin).

## ACP Expectations

Commit after each feature. Final commit message: `build: complete mini‑project`.

## (Optional) Publish

If time allows, enable **GitHub Pages** and include the live URL in your submission.

## Rubric (20 pts)

- **Semantics & Structure** (0–5) – Valid, minimal semantic regions used correctly.
- **Form & Accessibility** (0–5) – Labeled inputs; sensible attributes; ARIA used only where appropriate.
- **Flexbox Layout** (0–5) – Clear layout with responsive consideration.
- **Box Model & Overrides** (0–5) – Visible spacing/border and a UA‑override demonstrated.
