## Learning Goals

- Apply the **ACP** loop for each feature.
- Use **Live Server** and **Prettier** while coding.
- See a basic **merge conflict** and resolve it (instructor demo).

## Timeline

### 0–50 min - Demo 1: Start a tiny site, ACP each feature

- New repo (`simple-web-page`); `index.html`; basic structure; run Live Server.
- Feature commits you can model: `Add header`, `Add paragraph`, `Add simple styles`.

**10‑minute break**

### 60–110 min - Demo 2: Merge conflict (optional) & review

- Create `feature/a` and `feature/b` both editing the same line; open PR; resolve conflict in GitHub editor.
- Review ACP cadence one more time.

### 110–120 min - Breakout

- See `day-4-breakout-task.md`.
