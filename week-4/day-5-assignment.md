#### Objective
Interview on integrating Week 2 skills in one small page using semantic structure, an accessible form, Flexbox, and box model overrides.

#### Requirements
1. Live session with a TA or instructor.
2. Open your repo and run the page locally or show the live page if published.
3. Walk through the sections you built: header and nav, hero, content, CTA, footer.
4. Live changes (perform one or two as requested):
   - add or fix a form label and required attribute
   - add a minimal ARIA attribute only if it is needed
   - convert a block to a Flexbox layout or adjust an existing flex rule
   - add or tweak a mobile breakpoint for a clearer layout
   - demonstrate a visible box model change (padding, border, margin) on an element
   - override one user agent default such as paragraph margin
5. Answer brief questions about the choices you made and how they support readability and accessibility.

#### Submission
Public repo URL. If published, include the live URL.

#### Rubric (20 pts)
- Semantics and structure 0–5
- Form and accessibility 0–5
- Flexbox layout and responsive behavior 0–5
- Box model and UA override 0–5
